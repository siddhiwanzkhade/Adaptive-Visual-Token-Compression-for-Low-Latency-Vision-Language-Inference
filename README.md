# SwiftVLM :Adaptive-Visual-Token-Compression-for-Low-Latency-Vision-Language-Inference

> **Category:** Efficient Multimodal AI · VLM Inference Optimization · ML Systems  
> **Goal:** Reduce the number of visual tokens processed by a Vision-Language Model while preserving answer quality and improving latency, memory usage, and throughput.

---

## 1. Problem Statement

Vision-Language Models (VLMs) such as LLaVA, Qwen-VL, BLIP-2, and other multimodal assistants process images by converting them into many **visual patch tokens**. These visual tokens are then passed into a language model along with the user's text prompt.

This creates a major inference bottleneck.

As image resolution increases, the number of visual tokens also increases. The language model must attend over both text tokens and image tokens, which increases:

- time-to-first-token (TTFT)
- prefill latency
- GPU memory usage
- KV-cache size
- inference cost
- serving latency under batch/concurrent workloads

In production multimodal systems, not every visual patch is equally useful. Many image patches may contain background, redundant regions, or low-information content. Passing all visual tokens into the language model can waste computation.

### Core Problem

> Can we reduce the number of visual tokens passed into a VLM while preserving visual reasoning quality?

---

## 2. Proposed Solution

**VisCompress** is a lightweight visual token compression framework for efficient Vision-Language Model inference.

Instead of passing all image patch tokens into the language model, VisCompress compresses or selects the most informative visual tokens before multimodal reasoning.

```text
Input Image + Question
        ↓
Frozen Vision Encoder
        ↓
Visual Patch Tokens
        ↓
Visual Token Compression Module
        ↓
Compressed Visual Tokens
        ↓
Frozen / Partially Frozen VLM Decoder
        ↓
Answer
```

The project compares multiple token reduction strategies:

1. **Full-token baseline** — pass all visual tokens.
2. **Random token selection** — randomly keep K visual tokens.
3. **Mean pooling** — aggregate visual tokens into a smaller representation.
4. **Top-K token selection** — score and keep the most informative tokens.
5. **Learnable MLP compressor** — train a small adapter to compress tokens.
6. **Perceiver-style resampler** — use learned latent queries to produce a fixed number of compressed visual tokens.

The goal is to study the **quality-latency-memory tradeoff** of visual token compression.

---

## 3. Research Question

> Can visual tokens be reduced by **4x–8x** while preserving VQA/captioning quality and improving inference latency, memory usage, and throughput?

Example:

```text
Original visual tokens: 576
Compressed visual tokens: 128 / 64 / 32
Target: similar answer quality with lower inference cost
```

---

## 4. Why This Matters

Modern multimodal models are becoming increasingly expensive to serve. Even when the language model is small, image tokens increase context length and attention cost.

Visual token compression is useful for:

- faster visual question answering
- real-time image understanding
- accessibility systems
- mobile or edge multimodal AI
- low-cost VLM serving
- document/image understanding
- production multimodal chatbots
- resource-constrained inference

This project focuses on **inside-the-model efficiency**, not retrieval or RAG. The compression happens on visual patch tokens generated by the vision encoder.

## 9. Methods

### 9.1 Full-Token Baseline

Pass all visual tokens into the VLM.

This gives the reference quality, latency, and memory usage.

```text
576 tokens → VLM decoder
```

### 9.2 Random Token Selection

Randomly keep K visual tokens.

This is a weak baseline that checks whether intelligent selection is better than random pruning.

```text
576 tokens → random 64 tokens → VLM decoder
```

### 9.3 Mean Pooling

Aggregate visual tokens using average pooling.

This compresses information but may lose spatial detail.

```text
576 tokens → pooled representation → VLM decoder
```

### 9.4 Top-K Token Selection

Score each visual token and keep the top K tokens.

Scoring options:

- token activation norm
- attention score
- similarity to text/question embedding
- learned importance score

```text
576 tokens → score tokens → keep top 64 → VLM decoder
```

### 9.5 Learnable MLP Compressor

Train a lightweight MLP adapter that maps many visual tokens into fewer compressed tokens.

Only the compressor is trained; the vision encoder and language model remain frozen or mostly frozen.

```text
Visual tokens → MLP compressor → compressed visual tokens
```

### 9.6 Perceiver-Style Resampler

Use learned latent queries that attend over visual tokens and output a fixed number of compressed tokens.

This is the most technically advanced compression method in the project.

```text
576 visual tokens
        ↓
Perceiver Resampler
        ↓
64 compressed tokens
```

## 10. NLP / LLM Techniques Used

Although this is a multimodal systems project, it still uses several NLP and LLM techniques:

### Transformer Tokenization and Sequence Length Analysis

The project studies how visual tokens increase the effective context length processed by the language model.

### Attention and Context Compression

Visual token compression reduces attention workload by shortening the multimodal sequence.

### Frozen Encoder + Frozen Decoder Composition

The system follows a parameter-efficient multimodal design pattern:

```text
Frozen vision encoder + small trainable bridge + frozen language model
```

### Adapter-Based Learning

Instead of fine-tuning the full VLM, the project trains only a lightweight compressor or projector module.

### Prompt-Based VQA Evaluation

The VLM is prompted with image-question pairs and evaluated on answer correctness.

Example prompt:

```text
Question: What object is on the table?
Answer briefly.
```

### Structured Output Evaluation

Generated answers are normalized and compared against ground-truth answers using exact match or relaxed matching.

### Optional LLM-as-Judge Evaluation

For open-ended captions or ambiguous VQA answers, an LLM judge can be used to compare semantic equivalence between model output and reference answer.

### Hallucination Analysis

The project can measure whether aggressive token compression causes the model to hallucinate objects or miss key visual evidence.

## 13. Expected Outcome

The expected outcome is not simply “compression is better.” The project aims to identify the tradeoff curve.

Possible findings:

- moderate token compression improves latency with minimal quality loss
- very aggressive compression reduces quality
- Top-K selection outperforms random selection
- Perceiver-style resampling preserves quality better than simple pooling
- visual token reduction primarily improves prefill latency and memory, not necessarily decode latency
- benefits are larger at higher image resolutions or larger batch sizes

Target hypothesis:

> Reduce visual tokens from 576 to 64 while preserving most VQA quality and reducing prefill latency and peak GPU memory.

Final numbers should be reported only after experiments.
